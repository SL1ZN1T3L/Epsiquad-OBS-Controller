# Changelog

Все значимые изменения в проекте документируются в этом файле.

## [3.7.1] - 2026-05-10
### Добавлено
- Режим энергосбережения: при низком заряде (≤20% без зарядки) или системном режиме экономии приложение автоматически отключает анимации, опускает частоту экрана до 60Гц, реже опрашивает OBS (10с вместо 5с) и stats snapshot (30с вместо 10с), уменьшает FPS live-preview до 5 кадров/сек
- Настройка «Энергосбережение» в Settings: режимы Auto (по умолчанию) / Всегда вкл / Никогда, индикатор уровня заряда и текущего состояния
- Зависимость `battery_plus` для отслеживания заряда и системного режима экономии

### Исправлено
- Бесконечные попытки автопереподключения — добавлен лимит в 15 попыток, после чего попытки останавливаются с понятным сообщением и можно подключиться вручную
- Захват звука приложения и нестандартные аудио-источники теперь корректно определяются как звуковые: добавлен fallback-детектор через probe `GetInputMute`, который ловит источники с inputKind, не попадающим в стандартный список (например, плагины захвата звука приложения)
- В обратном отсчёте переподключения теперь отображается номер попытки в формате X/15
- Фантомное соединение при краше OBS — добавлен health-check: если 2 периодических запроса подряд падают по таймауту, соединение принудительно сбрасывается и запускается стандартный реконнект
- Полная синхронизация (fullSync) теперь корректно прерывается, если OBS не отвечает на первый запрос — раньше показывала «Готово», даже если все запросы падали по таймауту
- Рассинхрон UI с OBS после действий — добавлено пост-подтверждение через 300мс для критичных действий (mute/unmute, переключение сцены, видимость источника). UI корректируется, если реальное состояние OBS отличается от оптимистично применённого

### Улучшено
- Таймаут запросов теперь параметризуем (по умолчанию 15с), для post-verify используется 5с — фантомное соединение выявляется быстрее

## [3.7.0] - 2026-04-14
### Добавлено
- Система логов с возможностью отключения (Настройки → Диагностика), при выключении не расходует память
- Экран журнала событий с фильтрацией, поиском, автопрокруткой
- Действия с логами: копировать, сохранить в файл, поделиться, очистить
- Индикаторы громкости (Volume Meters) — OBS-стиль: зелёный→жёлтый→красный, плавное падение, пик-маркер
- Автообнаружение OBS в локальной сети (Auto Discovery) — сканирование подсети на порту 4455
- История стримов и записей с графиками FPS, CPU, RAM (fl_chart)
- Чат стрима — встроенный WebView для Twitch и YouTube с сохранением настроек
- Кастомные темы — 24 пресета в 4 категориях, hue-слайдер, чистый чёрный (AMOLED) режим, сохранение тем
- Конфигурация OBS: переключение коллекций сцен и профилей
- Управление переходами: выбор типа перехода, слайдер длительности с пресетами
- Управление фильтрами: просмотр и вкл/выкл фильтров любого источника
- Управление медиа-источниками: play/pause/stop/restart с прогресс-баром
- Меню инструментов (bottom sheet) вместо множества FAB-кнопок
- Живой обратный отсчёт при автопереподключении с номером попытки
- Автоматический сбор статистики каждые 10 секунд при активном стриме/записи

### Улучшено
- Все debugPrint заменены на структурированное логирование через LogService
- Optimistic updates для стрима/записи теперь откатываются при ошибке OBS
- Обнаружение медиа-источников через GetInputList по inputKind
- Навигация из bottom sheet использует сохранённый Navigator
- Подписка на события переходов, фильтров, медиа и Volume Meters (eventSubscriptions: 66047)
- Ползунок громкости отправляет значение в реальном времени при перетаскивании (throttle 80мс)
- Бэкап v2.0: сохраняет темы, профили Quick Control, чат, подтверждения, напоминания, логирование (12 ключей)
- Динамическая смена темы без перезапуска через ValueNotifier

### Исправлено
- _localTimeTimer не отменялся в dispose() (утечка таймера)
- OBSConnection поля сделаны final (immutable pattern)
- Защита от пайлапа запросов при обновлении статуса медиа-источников
- DropdownButtonFormField в фильтрах: проверка наличия значения в списке
- Volume Meter: использует post-fader peak (index 1), а не pre-fader inputPeak (index 2)
- Volume Meter: градиент привязан к позиции на треке, а не растягивается по заполнению

### Новые зависимости
- fl_chart — графики истории
- webview_flutter — чат стрима
